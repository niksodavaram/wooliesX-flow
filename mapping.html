A · MY 5 STORIES
Why these roles matter for this job
Scale, resilience, compliance

I’ve grouped 15+ years into anchor stories that match WooliesX’s problem space: running a large, regulated, distributed edge fleet that must keep trading when WAN, hardware or upstream systems fail.

Saab Australia — 9LV warship edge
Edge Infra Engineer · Nov 2024 – May 2025 · NV1 cleared · real-time systems
Edge infrastructure layer: RHEL + Ubuntu/Debian nodes connecting ship sensors (sonar, radar, AWDISSCNTRL) to HMI/tactical displays via RTI DDS middleware — OODA loop decision model. ISM/ASD controls, STIGs and CIS benchmarks on classified Linux stack with Ansible‑automated hardening. Offline‑first by design: a warship must maintain full capability when completely isolated from shore command. Git‑centred change and repeatable, compliant builds.
→ WooliesX: Offline warship = offline MicroShift store. RTI DDS = Kafka. ISM/ASD classified > PCI‑DSS. I have already built the harder version of the WooliesX problem.
NAB — 600+ branches AU‑wide
Senior Edge Fleet Analyst Engineer · Apr 2023 – Sep 2024
Managed ATMs, TCRs, card‑swipe, passbook printers and cash machines across ~600 branches. Orchestrated centrally via Rancher (hub+spoke Kubernetes fleet) over MPLS WAN, with F5 BIG‑IP, NGINX and HAProxy handling branch traffic. Kafka for AUSTRAC‑compliant event streaming; Ansible for fleet‑wide OS patching and CIS hardening across all branch edge nodes.
→ WooliesX: Rancher ≈ ACM · ~600 branches → 3,000 stores · MPLS WAN → SD‑WAN · AUSTRAC → PCI‑DSS. Same fleet problem, just larger scale.
Energy Australia — CDR platform
Greenfield Platform Engineer · Aug 2022 – Apr 2023
Built the ACCC‑mandated CDR platform from zero to production in 8 months on Azure/AKS. Designed and owned a hybrid control plane: central policy hub plus distributed clusters, architecturally identical to ACM + MicroShift. Policy‑as‑code gates and ACCC conformance testing in every pipeline; OAuth2/OIDC via Okta; zero compliance failures at go‑live.
→ WooliesX: AKS hybrid control plane = ACM + MicroShift. ACCC compliance mindset = PCI‑DSS discipline. Greenfield 8‑month delivery = practical migration playbook.
NovaStack — sovereign OpenStack / K8s
Principal Engineer · 2021 – 2022
Built a sovereign private‑cloud stack on OpenStack with Kubernetes on top. RHEL‑based images, Git‑driven configuration and OCP‑compatible manifests as a stepping stone toward OpenShift. Designed HA control planes, storage and network for regulated workloads needing strict data residency.
→ WooliesX: RHEL as the base everywhere, Kubernetes as the fabric and Git as the truth. Same opinionated, compliance‑friendly design you want at store edge.
CCG / Vic Uni — LEON Linux field stations
Distributed Edge Systems · PhD‑backed work
Built 16 LEON Linux base stations (Xilinx ML506 + Aeroflex Gaisler SPARC) across remote NZ/Alaska, tracking 3,000 tagged birds. Edge nodes ran local Bayesian inference and synced back via rsync/SSH when links came up, with Python/Bash/cron automation — zero on‑site support model.
→ WooliesX: Offline‑first Linux nodes that self‑heal and reconcile when the link returns — exactly the operating model for 3,000 MicroShift stores managed by ACM.
